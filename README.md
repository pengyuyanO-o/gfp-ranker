# GFP-Ranker: ESM2-Guided Fluorescence Prediction and Mutant Design

A complete protein engineering pipeline for predicting GFP fluorescence brightness from amino acid mutations and generating novel high-brightness candidates.

**Pipeline:** Raw mutation data → ESM2 embeddings → MLP ranker with pairwise ranking loss → candidate generation → top-candidate selection

## Background

We train a ranking model on ~141k experimentally measured GFP mutant–brightness pairs across four GFP scaffolds (avGFP, amacGFP, cgreGFP, ppluGFP). The model combines frozen ESM2 protein language model representations with physicochemical mutation features to predict Δlog(brightness). Novel multi-mutation candidates are generated by combining individually beneficial single-point mutations and scored with an ensemble of trained MLP models.

## Data Preparation

The pipeline expects:

1. **Mutation–brightness data** (Excel): columns `aaMutations`, `GFP type`, `Brightness`
   - `aaMutations` format: `WT` for wild-type; `V162A` for single mutation; `V162A:L177I` for combinations
   - Positions use **mature-protein 1-based numbering** (Met at position 0 excluded from numbering)

2. **Wild-type sequences** (text file): one sequence per GFP type, plain text or FASTA

Update `configs/default.yaml` with your data paths:
```yaml
paths:
  data_xlsx: /path/to/GFP_data.xlsx
  wt_seq_file: /path/to/wt_sequences.txt
```

## Step-by-Step Usage

### Step 1 — Data Preprocessing

```bash
python -m src.data     --config configs/default.yaml
python -m src.mutation --config configs/default.yaml
```

Parses mutations, applies them to WT sequences, validates each mutant, and saves `outputs/processed/gfp_processed.csv`.

### Step 2 — ESM2 Embedding Extraction (GPU required)

```bash
python -m src.embeddings_esm2 --config configs/default.yaml
```

Runs frozen ESM2-t33-650M (fp16) on all mutant sequences. Extracts per-token embeddings [L × 1280] and saves mean, WT-delta, and mutation-site aggregations as compressed `.npz` files (one per GFP type). Runtime: ~40 min for 141k sequences on one A6000.

### Step 3 — Feature Construction

```bash
python -m src.features --config configs/default.yaml
python -m src.splits   --config configs/default.yaml
```

Builds a 6,418-dimensional feature vector per sample and generates train/val/test splits.

### Step 4 — Model Training

```bash
# Baseline models (Ridge, ElasticNet, LightGBM)
python -m src.train_regression --config configs/default.yaml

# Main MLP ranker
python -m src.train_ranker --config configs/default.yaml
```

### Step 5 — Evaluation

```bash
python -m src.evaluate --config configs/default.yaml
```

### Step 6 — Candidate Generation & Selection

```bash
python -m src.select_top10 --config configs/default.yaml
```

Generates 10,000 novel multi-mutation candidates per GFP type, scores them with the MLP ensemble, and outputs ranked candidates.

## Algorithm Overview

### 1. Feature Engineering

Each sample is represented by a **6,418-dimensional** feature vector:

```
[mutant_mean | wt_mean | global_delta | mutsite_delta | mutsite_mutant]   (5 × 1280 = 6400 dims)
+ physicochemical_delta                                                    (13 dims)
+ single_mutation_prior                                                    (5 dims)
```

**ESM2 features (6,400 dims):**
- `mutant_mean`: mean over all token embeddings of the mutant sequence
- `wt_mean`: mean over all token embeddings of the WT sequence
- `global_delta`: mutant_mean − wt_mean (global sequence-level change)
- `mutsite_delta`: mean of (mutant − WT) at mutation-site token positions only (local effect)
- `mutsite_mutant`: mean of mutant embeddings at mutation-site positions only

ESM2 encodes evolutionary and structural context without any task-specific fine-tuning. The delta features isolate the local effect of each mutation from the global sequence context.

**Physicochemical features (13 dims):** Σ and mean of Δhydrophobicity (Kyte-Doolittle), Δcharge, Δmolecular weight; counts of proline/glycine/aromatic substitutions.

**Single-mutation prior (5 dims):** sum, mean, max, min, and coverage ratio of experimentally measured single-point mutation effects from training data — provides an additive approximation of multi-mutation effects.

### 2. MLP Ranker Architecture

```
Input (6418)
  → Linear(512) → LayerNorm → GELU → Dropout(0.2)
  → Linear(256) → GELU → Dropout(0.2)
  → Linear(1)
  → pred_Δlog_brightness
```

**Combined loss:**

```
L_total = HuberLoss(pred, target) + λ · L_rank

L_rank = Σ_{(i,j): target_i > target_j, same GFP type}
         w_ij · log(1 + exp(−(pred_i − pred_j)))

w_ij = |target_i − target_j| × (2 if either sample is in top-20% brightness, else 1)
```

The pairwise ranking loss enforces correct relative ordering within each GFP type. Top-20% sample pairs receive double weight, concentrating learning on the high-brightness regime critical for protein engineering. λ = 0.5.

**Training:** AdamW optimizer, CosineAnnealingLR schedule, early stopping (patience=12), batch size=256, 3 independent random seeds.

### 3. Brightness–Stability Balance

Pure brightness optimization risks generating sequences with poor foldability or structural integrity. We address this through four complementary mechanisms without requiring a separate stability model:

| Mechanism | Details |
|-----------|---------|
| **Ensemble uncertainty penalty** | `final_score = pred_mean − 0.5 × pred_std` across 3 MLP seeds. High inter-seed variance signals that a candidate lies outside the training distribution and is penalized directly in the selection score. |
| **Heuristic stability filter** | Candidates with >5 mutations, ≥2 proline introductions, or ≥2 charge reversals are flagged `stability_risk=caution`. Proline constrains backbone flexibility; charge reversals disrupt electrostatic networks. |
| **Single-mutation prior coverage** | Only candidates where all component single-mutations have measured training data (`single_prior_available_ratio = 1.0`) are selected, ensuring each mutation's effect is empirically grounded rather than extrapolated. |
| **Mutation count constraint** | Candidate generation is restricted to 2–5 mutations. Fewer mutations reduce epistatic complexity and experimental validation cost. |

> **Note:** `stability_risk` is a heuristic proxy, not a true thermodynamic stability predictor. For stricter filtering, combine with FoldX ΔΔG calculations or AlphaFold2 pLDDT scores.

### 4. Candidate Generation

For each GFP type:
1. Identify beneficial single-point mutations (training-set, top-20% brightness quantile)
2. Randomly sample combinations of 2–5 mutations (fixed seed=42 for reproducibility), enforcing that each position appears at most once per candidate
3. Generate 10,000 unique candidates per GFP type (candidates absent from training data)
4. Extract ESM2 embeddings for all 40,000 candidates
5. Score with 3-seed MLP ensemble; compute `final_score = pred_mean − 0.5 × pred_std`
6. Apply position-set diversity filter: remove candidates with identical mutation position sets
7. Select top candidates per GFP type

## Model Performance

Evaluated on held-out test sets across three split strategies:

### MLP Ranker (random stratified split, n=21,172)

| Seed | Spearman ↑ | NDCG@10 ↑ | Hit@10 ↑ |
|------|-----------|---------|--------|
| seed0 | 0.840 | 2.00 | 0.50 |
| seed1 | 0.829 | 2.39 | 0.40 |
| seed2 | 0.829 | **15.28** | **0.90** |

### Best Baseline per Split Strategy

| Model | Split | Spearman ↑ | NDCG@10 ↑ | Hit@10 ↑ |
|-------|-------|-----------|---------|--------|
| LightGBM | leave-position-out | **0.908** | 11.18 | 0.40 |
| LightGBM | mutation-count | 0.895 | 6.63 | **0.90** |
| Ridge | mutation-count | 0.851 | 25.01 | 0.10 |

**Metric definitions:**
- **Spearman**: rank correlation between predicted and true brightness across all test samples
- **NDCG@10**: Normalized Discounted Cumulative Gain at rank 10 — measures quality of top-10 ranking
- **Hit@10**: fraction of predicted top-10 samples that are truly in the top-10% brightness
- **Leave-position-out Spearman = 0.908** confirms the model generalizes to mutation positions not seen during training

### Candidate CSV Columns

| Column | Description |
|--------|-------------|
| `GFP_type` | Source GFP scaffold |
| `aaMutations` | Mutation string (e.g., `V162A:L177I`) |
| `mutant_sequence` | Full amino acid sequence |
| `mutation_count` | Number of point mutations |
| `pred_delta_log_brightness_mean` | Mean predicted Δlog(brightness) vs WT (3-seed ensemble) |
| `pred_delta_log_brightness_std` | Prediction uncertainty (std across 3 seeds) |
| `pred_log_brightness` | Predicted absolute log brightness |
| `final_score` | `pred_mean − 0.5 × pred_std` (primary selection criterion) |
| `single_prior_sum` | Sum of individual single-mutation training effects |
| `single_prior_available_ratio` | Fraction of mutations with training-data coverage |
| `stability_risk` | Heuristic stability flag (`low` / `caution:reason`) |

## Configuration

Key parameters in `configs/default.yaml`:

```yaml
paths:
  esm2_weight: pretrained/esm2_t33_650M_UR50D.pt

environment:
  cuda_visible_devices: "0"

features:
  use_esm2: true
  use_physchem: true
  use_single_prior: true

training:
  epochs: 80
  lr: 0.001
  lambda_rank_list: [0.5]   # ranking loss weight λ
  seeds: [0, 1, 2]          # ensemble seeds

top10:
  beneficial_single_top_quantile: 0.2   # top-X% brightness → beneficial mutation pool
  min_combination_order: 2
  max_combination_order: 5
  max_candidates_per_gfp_type: 10000
  uncertainty_beta: 0.5                 # β in final_score = mean − β·std
  final_topk: 10
```
